## 🛠️ Основні практичні кейси та інженерні рішення

### ⚙️ Кастомний сервіс моніторингу (Module 4)
Створено та інтегровано в дерево ініціалізації демон `myscript.service`, який керує Bash-скриптом для щосекундної фіксації міток часу у системному лозі. Налаштовано автоматичний перезапуск (`Restart=always`) та прив'язку до стадії завантаження системи `multi-user.target`.

### 🔑 Безпарольна архітектура SSH (Module 5)
Реалізовано схему автентифікації на основі криптографічних ключів **Ed25519**. Налаштовано локальний конфігураційний файл клієнта `~/.ssh/config` з абстракцією параметрів підключення:
```text
Host myserver
    HostName 127.0.0.1
    User ernest
    IdentityFile ~/.ssh/id_ed25519
```
Це дозволило виконувати прокидання файлів через `scp` та синхронізацію директорій через `rsync` в один клік без ручного введення реквізитів доступу.

### 🛡️ Надійний Bash-скрипт архівації (Module 6)
Розроблено production-ready скрипт `backup.sh` для пакування логів, який містить:

* Сувору валідацію вхідних параметрів та перевірку існування каталогів.

* Захист від Race Conditions (паралельного запуску) через механізм створення Lock-файлу у `/tmp/backup.lock`.

* Перехоплення системних сигналів (`trap` на подію `EXIT`), що гарантує очищення тимчасових ресурсів навіть при аварійному завершенні (`Ctrl+C`).

### 🐳 Глибоке розуміння Docker PID 1 (Module 7)
Проведено аналіз ізоляції процесів (Namespaces) всередині контейнера `Nginx`. Досліджено поведінку системи під час зупинки контейнера:

* Передача сигналу `SIGTERM` (Signal 15) для graceful shutdown.

* Автоматичний таймаут (10с) та механізм примусової зупинки через ядро за допомогою `SIGKILL` (Signal 9).

* Концепція логування Twelve-Factor App (перехоплення Docker-рушієм потоків `stdout`/`stderr` головного процесу).

## 💻 Технологічний стек середовища розробки
* ОС Хоста: Windows 10 / 11 Pro

* Підсистема: WSL2 (Hyper-V виділена архітектура віртуалізації)

* Гостьова ОС: Ubuntu 26.04 LTS / Alpine Linux (всередині Docker)

* Контейнеризація: Docker Engine (Native-встановлення в підсистемі)

Всі матеріали підготовлено в рамках навчального треку з системної інженерії.
